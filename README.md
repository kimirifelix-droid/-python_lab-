Part A - Project Setup Using the CLI

I created the project directory in my home folder, then used the command line to make the required subfolders and files. The command `mkdir` created the folders, `New-Item` created the Python files, and `Set-Content` wrote the README content. This sequence established the basic project layout in a simple and repeatable way.

Separating code into `src`, `tests`, and `docs` keeps a project organized even when it is small. The `src` folder holds the main program code, the `tests` folder is where validation and future checks can live, and the `docs` folder stores notes and documentation. This structure makes it easier to understand, maintain, and expand the project later.

 Part B - Git Initialization and First Commit

I initialized a Git repository with `git init`, created a `.gitignore` file to exclude generated files, and used `git add` and `git commit` to save the initial state. The `.gitignore` file prevents unnecessary or sensitive files from being tracked, so the repository stays clean and focused on source files.

The `.gitignore` entries matter because `__pycache__/` and `*.pyc` are generated by Python during execution and do not need to be stored in version control, while `.env` files may contain secrets that should not be shared. The commit history shows the development timeline of the project by recording each meaningful change in order, which helps others and future collaborators understand how the project evolved.

 Part C - Writing and Committing Python Code

# utils.py
```python
def square(n):
    return n * n


def is_even(n):
    return n % 2 == 0


def celsius_to_fahrenheit(c):
    return (c * 9 / 5) + 32


def greet(name):
    return f"Hello, {name}!"
```

# main.py
```python
from utils import square, is_even, celsius_to_fahrenheit, greet


def main():
    user_input = input("Enter a number: ")
    number = float(user_input)

    print(f"Square: {square(number)}")
    print(f"Even: {is_even(number)}")
    print(f"Fahrenheit: {celsius_to_fahrenheit(number)}")
    print(greet("student"))


if __name__ == "__main__":
    main()
```

Python's import system connects `main.py` to `utils.py` by allowing `main.py` to load functions from the module file `utils.py` using the `from utils import ...` statement. When Python runs `main.py`, it looks for `utils.py` in the same package or folder path and makes those functions available so they can be called directly.

 Part D - Publishing to GitHub and Branch Workflow

The project was prepared for GitHub publication by creating a repository remote and using Git branches for the feature work. A feature branch named `feature/add-greeting` was created so the greeting enhancement could be developed separately from the main branch before being merged.

The branch workflow is useful because it keeps the main line of development stable while allowing new features or experiments to be worked on safely. When the change is complete, a pull request can be opened to review and merge the feature into the main branch.
